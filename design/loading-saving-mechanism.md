# 游戏世界加载和保存机制设计

## 1. 加载机制

### 1.1 世界加载流程

1. **初始化加载器**：
   - 创建 `WorldLoader` 实例，设置世界目录路径
   - 初始化缓存系统

2. **加载世界配置**：
   - 读取 `world.json` 文件
   - 解析 JSON 数据，创建 `GameWorld` 实例
   - 验证世界配置数据的完整性

3. **加载区域**：
   - 遍历 `regions` 目录
   - 对每个区域目录，调用 `load_region()` 方法

4. **加载全局NPC**：
   - 遍历 `global_npcs` 目录
   - 对每个NPC文件，调用 `load_npc()` 方法

5. **加载全局物品**：
   - 遍历 `global_items` 目录
   - 对每个物品文件，调用 `load_item()` 方法

6. **加载任务**：
   - 遍历 `tasks` 目录
   - 对每个任务文件，调用 `load_task()` 方法

7. **构建索引**：
   - 为世界中的所有实体构建索引，加速查找
   - 建立房间之间的连接关系

### 1.2 区域加载流程

1. **加载区域配置**：
   - 读取 `region.json` 文件
   - 解析 JSON 数据，创建 `Region` 实例

2. **加载房间**：
   - 遍历 `rooms` 目录
   - 对每个房间文件，调用 `load_room()` 方法

3. **加载NPC**：
   - 遍历 `npcs` 目录
   - 对每个NPC文件，调用 `load_npc()` 方法

4. **加载物品**：
   - 遍历 `items` 目录
   - 对每个物品文件，调用 `load_item()` 方法

### 1.3 房间加载流程

1. **加载房间配置**：
   - 读取 `room.json` 文件
   - 解析 JSON 数据，创建 `Room` 实例

2. **处理出口**：
   - 解析房间出口数据
   - 建立与目标房间的引用关系

3. **加载NPC**：
   - 从房间配置中读取NPC列表
   - 对每个NPC，调用 `load_npc()` 方法

4. **加载物品**：
   - 从房间配置中读取物品列表
   - 对每个物品，调用 `load_item()` 方法

### 1.4 实体加载流程

1. **NPC加载**：
   - 读取NPC JSON文件
   - 解析数据，创建 `Npc` 实例
   - 初始化NPC的行为模式和提示词系统

2. **物品加载**：
   - 读取物品 JSON文件
   - 解析数据，创建 `Item` 实例
   - 初始化物品的效果和属性

3. **任务加载**：
   - 读取任务 JSON文件
   - 解析数据，创建 `Task` 实例
   - 初始化任务的目标和奖励

## 2. 保存机制

### 2.1 世界保存流程

1. **初始化保存器**：
   - 创建 `WorldSaver` 实例，设置世界目录路径
   - 确保目录结构存在

2. **保存世界配置**：
   - 将 `GameWorld` 实例序列化为 JSON
   - 写入 `world.json` 文件

3. **保存区域**：
   - 对每个区域，调用 `save_region()` 方法

4. **保存全局NPC**：
   - 对每个全局NPC，调用 `save_npc()` 方法

5. **保存全局物品**：
   - 对每个全局物品，调用 `save_item()` 方法

6. **保存任务**：
   - 对每个任务，调用 `save_task()` 方法

### 2.2 区域保存流程

1. **保存区域配置**：
   - 将 `Region` 实例序列化为 JSON
   - 写入 `region.json` 文件

2. **保存房间**：
   - 对每个房间，调用 `save_room()` 方法

3. **保存NPC**：
   - 对每个NPC，调用 `save_npc()` 方法

4. **保存物品**：
   - 对每个物品，调用 `save_item()` 方法

### 2.3 房间保存流程

1. **保存房间配置**：
   - 将 `Room` 实例序列化为 JSON
   - 写入 `room.json` 文件

2. **保存NPC**：
   - 对每个房间内的NPC，调用 `save_npc()` 方法

3. **保存物品**：
   - 对每个房间内的物品，调用 `save_item()` 方法

### 2.4 实体保存流程

1. **NPC保存**：
   - 将 `Npc` 实例序列化为 JSON
   - 写入对应的NPC文件

2. **物品保存**：
   - 将 `Item` 实例序列化为 JSON
   - 写入对应的物品文件

3. **任务保存**：
   - 将 `Task` 实例序列化为 JSON
   - 写入对应的任务文件

## 3. 错误处理

### 3.1 加载错误处理

1. **文件不存在**：
   - 对于非关键文件，使用默认值
   - 对于关键文件，抛出错误并停止加载

2. **JSON解析错误**：
   - 记录错误信息
   - 尝试使用备份文件（如果存在）
   - 否则使用默认值或停止加载

3. **数据验证错误**：
   - 记录错误信息
   - 修复或忽略无效数据
   - 确保加载过程继续

### 3.2 保存错误处理

1. **目录不存在**：
   - 自动创建目录结构

2. **文件写入失败**：
   - 尝试使用临时文件，然后重命名
   - 记录错误信息
   - 继续保存其他数据

3. **磁盘空间不足**：
   - 检测磁盘空间
   - 记录错误信息
   - 停止保存过程

## 4. 缓存机制

### 4.1 内存缓存

1. **实体缓存**：
   - 缓存已加载的实体，避免重复加载
   - 使用 LRU 策略管理缓存大小

2. **房间缓存**：
   - 缓存最近访问的房间
   - 预加载相邻房间

3. **区域缓存**：
   - 缓存当前活动区域
   - 懒加载非活动区域

### 4.2 磁盘缓存

1. **序列化缓存**：
   - 缓存序列化后的实体数据
   - 减少重复序列化开销

2. **索引缓存**：
   - 缓存实体索引
   - 加速实体查找

## 5. 性能优化

### 5.1 加载优化

1. **并行加载**：
   - 使用多线程并行加载不同区域
   - 利用异步I/O提高加载速度

2. **懒加载**：
   - 只加载当前需要的区域和房间
   - 延迟加载未使用的实体

3. **预加载**：
   - 预加载可能很快需要的区域和房间
   - 利用空闲时间加载数据

### 5.2 保存优化

1. **增量保存**：
   - 只保存修改过的实体
   - 减少I/O操作

2. **批量保存**：
   - 批量写入多个文件
   - 减少磁盘寻道时间

3. **异步保存**：
   - 在后台线程中执行保存操作
   - 不阻塞游戏主线程

## 6. 安全性考虑

### 6.1 数据验证

1. **输入验证**：
   - 验证所有加载的数据
   - 防止恶意数据导致游戏崩溃

2. **数据一致性**：
   - 确保保存的数据与内存中的数据一致
   - 避免数据损坏

### 6.2 备份机制

1. **自动备份**：
   - 定期创建世界数据的备份
   - 保存多个版本的备份

2. **恢复机制**：
   - 提供从备份恢复的功能
   - 处理数据损坏的情况

## 7. 实现细节

### 7.1 文件操作

1. **文件路径管理**：
   - 使用相对路径构建文件路径
   - 处理不同操作系统的路径分隔符

2. **文件读写**：
   - 使用缓冲I/O提高性能
   - 处理文件锁，避免并发访问冲突

### 7.2 序列化与反序列化

1. **JSON处理**：
   - 使用高效的JSON库
   - 处理大文件的序列化和反序列化

2. **数据转换**：
   - 实现实体与JSON之间的转换
   - 处理复杂的数据结构

### 7.3 索引与查找

1. **实体索引**：
   - 使用哈希表存储实体ID到实体的映射
   - 支持快速查找实体

2. **空间索引**：
   - 构建房间的空间索引
   - 支持基于位置的查找

## 9. 总结

本设计文档详细描述了游戏世界的加载和保存机制，包括加载流程、保存流程、错误处理、缓存机制、性能优化和安全性考虑等方面。通过这些机制，可以实现游戏世界数据的高效加载和保存，同时确保数据的安全性和一致性。
